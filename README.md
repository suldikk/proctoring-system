# Система онлайн-прокторинга

Учебный проект на Java для демонстрации онлайн-экзамена с прокторингом. В системе есть авторизация студента и проктора, открытие доступа к экзамену, проверка камеры/микрофона, браузерное расширение, контрольный снимок студента, автоматический анализ камеры, журнал событий, снимки камеры и фрагменты записи.

## Возможности

- Авторизация через JWT.
- Роли пользователей: студент, проктор, администратор.
- Панель проктора с открытием доступа к экзамену.
- Страница студента с тестом, таймером, камерой и проверкой расширения.
- Проверка камеры и микрофона перед экзаменом.
- Контрольный снимок лица студента перед началом экзамена.
- Детекция нарушений:
  - выход из полноэкранного режима;
  - новая вкладка или потеря фокуса;
  - отсутствие лица в кадре;
  - больше одного лица;
  - телефон в кадре;
  - попытка копирования текста теста;
  - попытка скриншота/печати страницы.
- Затемнение экрана, если лицо студента пропало из кадра.
- Автоматический провал экзамена при обнаружении телефона.
- Плановые снимки камеры и 10-секундные фрагменты записи.
- Журналы аудита действий пользователей.
- Docker, Docker Compose, Kubernetes-манифесты, GitHub Actions и k6.

## Архитектура

Проект построен по слоистой архитектуре:

```text
Controller -> Service -> Repository -> Domain
     |           |             |
     v           v             v
    DTO       Mapper      JPA Entity
```

Основные пакеты:

- `controller` - REST API и обработка HTTP-запросов.
- `service` - бизнес-логика, проверки доступа, аудит, работа с медиа.
- `repository` - доступ к данным через Spring Data JPA.
- `repository.entity` - JPA-сущности базы данных.
- `domain` - роли, статусы сессий, типы событий.
- `dto` - объекты запросов и ответов API.
- `mapper` - преобразование Entity в DTO.
- `security` - JWT, фильтр аутентификации, роли.

## Локальный запуск

### Быстрый запуск демо

Для демонстрации без PostgreSQL можно использовать профиль `local`. В этом режиме используется H2 in-memory база данных.

```bash
mvn spring-boot:run "-Dspring-boot.run.profiles=local"
```

Если порт `8080` занят:

```bash
mvn spring-boot:run "-Dspring-boot.run.profiles=local" "-Dspring-boot.run.arguments=--server.port=8081"
```

### Запуск с инфраструктурой

Поднять PostgreSQL, Redis и Kafka:

```bash
docker compose up -d
```

Запустить приложение:

```bash
mvn spring-boot:run
```

Проверить состояние:

```bash
curl http://localhost:8080/actuator/health
```

## Сценарий демонстрации

1. Запустить приложение с профилем `local`.
2. Установить расширение из папки `browser-extension`.
3. Открыть панель проктора: `http://localhost:8080/proctor-dashboard/`.
4. Войти как проктор.
5. Нажать **Открыть доступ к экзамену**.
6. Открыть страницу студента: `http://localhost:8080/exam-demo/`.
7. Войти по ИИН и паролю студента.
8. Разрешить камеру и микрофон.
9. Сделать контрольный снимок лица.
10. Нажать **Я запустил расширение**.
11. После успешной проверки расширения экзамен начнётся автоматически.
12. Проверить нарушения:
    - убрать лицо из кадра;
    - открыть новую вкладку;
    - выйти из полноэкранного режима;
    - показать телефон в камеру.

При обнаружении телефона экзамен останавливается и считается проваленным. У проктора появляется событие и снимок нарушения.

## Поведение студента

Студент видит только:

- тест;
- таймер;
- своё изображение с камеры;
- ФИО;
- группу;
- дисциплину;
- количество вопросов;
- предупреждения во время экзамена.

Студент не видит поток событий, количество нарушений, снимки и записи.

Перед экзаменом студент проходит проверки:

1. Авторизация.
2. Проверка доступа, который открывает проктор.
3. Проверка камеры и микрофона.
4. Контрольный снимок лица.
5. Проверка браузерного расширения.
6. Автоматический старт экзамена.

## Поведение проктора

Проктор видит:

- назначенные сессии;
- кнопку открытия доступа к экзамену;
- контрольный снимок студента;
- поток событий;
- общее количество нарушений;
- снимки камеры;
- фрагменты записи.

Данные на панели проктора обновляются автоматически.

При выходе проктора из аккаунта:

- доступ к экзамену закрывается;
- события, снимки и фрагменты записи очищаются для следующей демонстрации.

## Снимки и запись

Во время экзамена система сохраняет:

- первый снимок камеры при старте экзамена;
- плановый снимок камеры каждые 5 минут;
- первый 10-секундный видео-фрагмент при старте экзамена;
- 10-секундный видео-фрагмент каждые 5 минут;
- снимок нарушения при обнаружении телефона.

Файлы сохраняются по умолчанию в:

```text
storage/proctoring
```

Путь можно изменить переменной окружения:

```bash
PROCTORING_STORAGE_ROOT=/path/to/proctoring-storage
```

## Расширение браузера

MVP расширения находится в папке:

```text
browser-extension
```

Установка в Google Chrome:

1. Открыть `chrome://extensions/`.
2. Включить режим разработчика.
3. Нажать **Загрузить распакованное расширение**.
4. Выбрать папку `browser-extension`.
5. Открыть страницу экзамена.

Если расширение не запущено, студент не сможет перейти к экзамену.

## Примеры API

### Регистрация

```http
POST /api/auth/register
Content-Type: application/json

{
  "email": "student@example.com",
  "password": "password123",
  "fullName": "Студент Один",
  "roles": ["STUDENT"]
}
```

### Вход

```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "proctor@example.com",
  "password": "password123"
}
```

### Создание экзаменационной сессии

```http
POST /api/sessions
Authorization: Bearer <token>
Content-Type: application/json

{
  "examTitle": "Итоговый экзамен по Java",
  "studentId": "00000000-0000-0000-0000-000000000000",
  "proctorId": "00000000-0000-0000-0000-000000000001",
  "startsAt": "2026-05-01T09:00:00Z",
  "endsAt": "2026-05-01T11:00:00Z"
}
```

### Открытие доступа к экзамену

```http
PATCH /api/sessions/{sessionId}/status/ACTIVE
Authorization: Bearer <token>
```

### Создание события прокторинга

```http
POST /api/sessions/{sessionId}/events
Authorization: Bearer <token>
Content-Type: application/json

{
  "type": "TAB_SWITCH",
  "severity": 5,
  "details": "Студент открыл новую вкладку"
}
```

## Безопасность и конфиденциальность

Реализовано:

- JWT-аутентификация;
- RBAC через роли `STUDENT`, `PROCTOR`, `ADMIN`;
- проверки доступа к сессиям по роли и участнику экзамена;
- хеширование паролей;
- аудит действий пользователей;
- очистка прокторинговых артефактов после выхода проктора;
- разделение интерфейса студента и проктора.

Частично реализовано:

- ABAC-подход через проверки владельца/участника сессии;
- privacy-oriented подход за счёт ограничения доступа к медиа и очистки данных.

Не реализовано полностью:

- OAuth2;
- полноценное шифрование данных PostgreSQL на уровне БД;
- полноценное GDPR-соответствие;
- live streaming через WebRTC Media Server.

## Процесс разработки на GitHub

Используемые ветки:

- `main` - стабильная версия;
- `dev` - ветка разработки;
- `feature/*` - отдельные функции.

Примеры коммитов:

- `feat: add camera proctoring demo`
- `fix: secure registration flow and add auth tests`
- `feat: polish proctoring exam demo`

Рекомендуемый процесс:

1. Создать ветку `feature/...`.
2. Сделать коммиты с тегами `feat`, `fix`, `refactor`, `test`, `docs`, `ci`.
3. Открыть Pull Request.
4. Дождаться проверок GitHub Actions.
5. Влить изменения в `dev` или `main`.
